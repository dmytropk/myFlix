Goal is to build the server­side component of a "movies" web application

The REST API does the following:

  • Return a list of ALL movies to the user
  • Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user
  • Return data about a genre (description) by name/title (e.g., “Thriller”)
  • Return data about a director (bio, birth year, death year) by name
  • Allow new users to register
  • Allow users to update their user info (username, password, email, date of birth)
  • Allow users to add a movie to their list of favorites
  • Allow users to remove a movie from their list of favorites
  • Allow existing users to deregister

Please check the next table as a reference of what to expect for the "body" data in each request and response

Request URL HTTP Method Request body data format Response body data format
Get a list of movies /movies GET None A JSON object holding data about all the movies
Get data about a single movie by title /movies/[title] GET None A JSON object holding data about a single movie, containing a title, description, genre, director, image URL:

{
   title: 'Fight Club',
   description: 'Fight Club is a 1999 film based on the 1996 novel of the same name by Chuck Palahniuk',
   genre: 'Drama',
   directed: 'David Fincher',
   imageURL: '#'
}

Get data about a genre by title /movies/[title]/genre GET None A text message with the genre of a movie title
Get data about a director by name /directed/[name] GET None A JSON object holding data about a director or directors (includes bio, birth year, death year)
Allow new users to register /users POST A JSON object holding data about the user to add. Example:

{
   username: 'JohnDoe',
   email: 'johndoe@email.com',
   password: 'swordfish',
   dob: '01/01/2001',
   favorites: []
}

A JSON object holding data about the user that was added (includes id). Example:

{
   id: 1,
   username: 'JohnDoe',
   email: 'johndoe@email.com',
   password: 'swordfish',
   dob: '01/01/2001',
   favorites: []
}

Allow users to update their info /users/[id]/[username or email, or password, or dob] PUT None A text message indicating whether the user's information was successfully updated
Allow users to add a movie to their list of favorites /users/[username]/favorites POST A JSON object holding data about the user A JSON object holding data about the movie that was added to user's favorites:

{
   id: 1,
   username: 'JohnDoe',
   email: 'johndoe@email.com',
   password: 'swordfish',
   dob: '01/01/2001',
   favorites: ['Fight Club']
}

Allow users to remove a movie from their list of favorites /users/[username]/favorites DELETE None A text message indicating whether the movie was successfully removed from favorites list
Allow existing users to deregister /users/[id] DELETE None A text message indicating whether the user was successfully deregistered
